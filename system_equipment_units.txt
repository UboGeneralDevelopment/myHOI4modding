
装備とユニット解説

最低限はunitフォルダ内のunitとequipmentを書けばよい。テスト時にいろいろヘマして余計なことを考えすぎた。
equipmmentの導入はequipment、必要ならtechnology、あとtech.gfxを書けば画像付きの完全版ができる。
unitを導入するとなるとこれに加えてunitを書いて、あとunit_icon.gfxも書く必要ある。
新equipmentおよび新ユニットの導入について検討が必要。
装備と人的補給速度については現状一律変化のみ可能で、在庫があればあっという間に補給されていく。

ユニットの基本的なステータスは装備とユニットの組み合わせによって決まる。
ユニットと装備のstatsは多彩であり、通常はそれぞれのファイルに必要なものだけ記載する。
最終的にはユニットのstatsおよびユニット補正のかけられた装備のstatsの合計がユニットの最終statsになる。
この最終statsのうち、装備に由来するものは充足率に比例する。ユニットに由来するstatsは充足率関係ない。
つまり充足がくそだろうと士気の高いユニットは意外に粘れるということである。



陸軍ユニット
陸軍ユニットはstatsがequipmentとunitによって決まる。
まず、battalionのstatsがequipmentとunitによって決まり、その後divisionのstatsがbattalionのstatによって決まる。
battalionは普通は１種類のequipmentしか持たないが、2種類以上のequipmentを持つ場合は合計値が適用される。
ただし、transportが設定すると速度はtransportのequipmentによって決まる。

equipment stats
基本値がequipmentで決まるもの。unitのmodifiersによっても補正される。
師団statは大隊のstatの合計、最大値、最低値、平均、重みづけ平均などになる。

#コスト等
build_cost_ic = <float> #生産コスト　合計
resources = { ~~~ } #必要資源　合計
lend_lease_cost = <float> #レンドリース時必要輸送船

#以下の値は充足率に依存しない
reliability = <float> #信頼性　平均
maximum_speed = <float> #最高速度　最低値

armor_value = <float> #装甲値　重みづけ平均
ap_attack = <float> #貫通　重みづけ平均

#以下の値は装備充足率依存
defense = <float> #防御値　合計
breakthrough = <float> #突破値　合計
soft_attack = <float> #対人攻撃　合計
hard_attack = <float> #対装甲攻撃　合計
air_attack = <float> #対空　合計
hardness = <float> #装甲化率　平均？


unit stats, unit modifiers
主にユニットによって決まるステータス。また、ユニットごとにequipmentにmodifiersを与えることもある。
師団statは大隊のstatの合計、最大値、最低値、平均、重みづけ平均などになる。

#基本stat
max_organisation = <float> #最大組織率　平均
default_morale = <float> #組織率回復率　平均
max_strength = <float> #最大耐久値　合計
combat_width = <int> #戦闘幅　合計

#維持関係
weight = <float> #船で運ぶときの重量　合計
supply_consumption = <float> #補給消費　合計　
manpower = <int> #必要人的資源　合計
training_time = <int> #訓練時間　最大値

#師団に直接影響する補助能力
entrenchment = <float> ＃塹壕　多分合計
recon = <float> # 偵察値　多分合計

ユニットによる装備補正
装備のstatに掛け算で適用可能。補正は掛け算でかかる。特殊なものとしてattackやdefenceはsoft,hard,defence,breakthroughの両方にかかる
どのmodifierが適用可能なのかはものによるため注意が必要。おそらく技術による補正はここにかかっているので技術補正がかかっているものならここで使える。

ユニットによる地形補正
特定地形におけるユニットおよび装備のstatを変更可能。補正は掛け算でかかる。
バニラではattack,defence,movementしか使っていない。


ユニット、技術、アイデアによって与えることができるモディファー


ユニットや装備の分類はいろいろある。それぞれ以下のようにわけられる。カテゴリー以外の分類はmodでは増やせない。

タイプ
ユニットおよび装備の分類。
複数のタイプに所属可能。ユニットのタイプと、装備のタイプは基本的には一致するが何とも言えない。
自動車歩兵などは歩兵タイプではないが歩兵装備を使うのでよくわからん。
使い道としては、今のところアイデアによる装備ボーナスは装備のタイプが参照されることは判明している。
ユニットに設定されたタイプが何か効果あるかといわれると、よくわからん。
ユニットの装備補正は、装備している装備に対してタイプにかかわらず適用されるので、ユニットタイプの意味がよくわからない。

グループ
単一のグループにしか所属不可能。師団編成画面でのグループ選択に用いられ、種類は支援、歩兵、機動、装甲のみ

カテゴリー
ユニットの分類。技術による補正を与える際に用いる。複数カテゴリーに所属可能、技術補正はorで適用される。カテゴリーはmodで増やせる。
基本的に技術による補正は、直接装備statを補正するではなく、ユニットによる装備補正と、ユニットそのものへの補正を行う形となっている。
