加筆中

正直なところ、一番めんどくさいんではないかとさえ思えるシステム。
めんどくさいというのは、微妙に記述方法が統一されてなかったり、意味があるのかわからない部分があったりするあたりである。

アイデアは主には国レベルでのモディファーを与える。
アイデアによってもたらされる効果は非常に多く、一覧にすると以下の通り。
modifier ={} #MODシステム共通に使えるモディファー、種類は英語wikiのmodifierを参照。
equipment_bonus ={} #アイデア、ユニット、技術でつかえるボーナス。装備に対して補正を与える。
trait trait で指定された効果をもたらす
基本的には

アイデアの指定と識別方法はかなり複雑である。
第一に、一つ一つのアイデアは固有のidによって識別される。これの複製はエラーの原因となり避けるべきである。

次に、アイデアはhidden,national_spirit,それ以外のカテゴリー(バニラではgovernment,research,military)に分類される。
カテゴリーはさらにサブカテゴリーとして、スロットごとに分類することもできる。なお通常hiddenはhiddenカテゴリー、national_spiritはcountryカテゴリーである。
カテゴリーの定義の時に取得コストや解除コストなどの補正をかけられる。これはサブカテゴリーにも一括適用される。
通常は取得コストと解除コストのみ定義されている。また取得コストや解除コストは個別のアイデア定義において上書きすることが可能。

個別にアイデアを書く時には、まずカテゴリーかスロット分類してからアイデアの定義に入る。
lawやdesigner、national spiritは、カテゴリー分類あるいはスロット分類あるいは個別アイデアごとに指定することができる。下層の段階の指定が優先適用される。
これが定義されることによる効果は不明瞭。多分使用する画像の問題？

個別のアイデアは、まず入手性が2段階ある。allowedとavailableで、この2段階で国ごとにアクセスを制限している。
allowedはアイデアが見える条件を、availableはアイデアが取得可能になる条件を指定する。
countryやhiddenの場合はallowedはほぼ常にNOであり、そのほか国ごとに固有ならばトリガーに国家タグが使用される。共通の場合はたいていallowは書かれてない。
availableはallowの次に適用され、アイデアが取得可能になる条件が書かれる。
cancelは、条件によってアイデアを取り外すものである。

具体的な個別のアイデアの内容指定だが、まずは画像指定から。
アイデアの画像指定はアイデアのtxtファイルとgfxのファイルの双方によって決められている。
まず、アイデアidかpictureによって画像参照用のidが指定される。pictureがあればそれが画像参照用idであり、なければアイデアidがそのまま画像参照用idとなる
そしてGFXファイル内で、アイデアidまたはpictureでの画像参照用idと画像ファイルを結合する。
GFX_idea_~~~~という形であらわされており、それぞれについて参照すべき画像ファイルが指定されている。

問題は、バニラファイルの多くの国の閣僚などのアイデアではpicture指定がないいことである。このためコピペが面倒になっている。
そのまま丸ごとコピペすればid重複エラーが起き、アイデアidを書き換えれば、pictureが指定なしのために画像が表示されない。
コピペしたあとにidを書き換え、画像も変えたくなければ、pictureを書き加えなければならない。
ちなみにさらなる問題としては、localizationの問題がある。
localizationはアイデア中での指定はなく、アイデアidをlocalization側が参照するため、id書き換えたらlocalizationも新しいidに対応するものを書き加える必要がある。

ところで、元のアイデアを消し飛ばしてしまえばよいと考えることもあるかもしれないが、
NFやアイデアはあらゆるところのトリガーにかかわるため、消すとやばい量のエラーが出る。
正直言ってくそめんどくさい。

さて、具体的なアイデアの効果については、主に5種類ある。
modifiers
　modifiersを通常国レベルで与える。
equipment_bonus
　国レベルで装備品に補正を与えられる。ユニットへの補正ではなく、装備品への補正。装備品に効果を与えるのはtechnologyではできない。
補正は対象のequipmentに書かれたstatsならほぼすべて補正可能。おそらく補正量はかけ算で乗る。
researchbonus
　technologyカテゴリーごとにボーナスを与える。アイデアがある限り恒久的効果。
traits
　countryleaderのtraitsで定義されている。modifiersとequipmentbonusはこちらで定義可能。researchbonusは不明
on add
　加えたときにコマンドを発動可能。


例

XXX_example_idea = {
      designer = no			
      
      picture = generic_coastal_navy
      
      cost = 200
      removal_cost = 1000
      
      available = {
           has_government = fascism
      }
      
      allowed = {
           has_war = yes
      }
      		
      allowed_civil_war = {
           always = yes
      }
      on_add = {

     }
      
      modifier = {
           license_armor_purchase_cost = -0.5
      }
      
      research_bonus = {
           land_doctrine = -0.05
      }
      		
      equipment_bonus = {
           capital_ship = {
                attack = 0.1
           }
           screen_ship = {
                sub_detection = 0.1 build_cost_ic = -0.1
           }
           submarine = {
                sub_visibility = -0.1
           }
      }
      
      traits = { fascist_demagogue }
      
      ai_will_do = {
           factor = 1
      }
}
